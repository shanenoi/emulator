# v0.2 Learning Guide — Branches, Flags, and Loops

This guide explains the new ideas introduced in **v0.2 — Branches and Loops**.

v0.1 could only run straight-line code:

```asm
movz x0, #2
add  x1, x0, #3
hlt  #0
```

That kind of program always moves forward by one instruction at a time.

v0.2 adds control flow. The emulator can now decide where `pc` goes next.

## 1. The old v0.1 flow

In v0.1, most instructions did this:

```text
pc = pc + 4
```

ARM64 instructions are 4 bytes, so execution naturally moved like this:

```text
0x1000 -> 0x1004 -> 0x1008 -> 0x100c
```

This is called straight-line execution.

## 2. The new v0.2 idea: branches

A branch changes `pc` directly.

Instead of:

```text
pc = pc + 4
```

it can do:

```text
pc = target_address
```

Example:

```asm
b target
nop
target:
hlt #0
```

Execution starts at `b target`. The branch skips the `nop` and jumps directly to `target`.

Memory looks like this:

```text
0x1000: b target
0x1004: nop
0x1008: hlt #0
```

After the branch executes:

```text
pc = 0x1008
```

## 3. Labels are not stored in the binary

In assembly, labels are names for addresses:

```asm
loop:
    sub x0, x0, #1
    cbnz x0, loop
```

The emulator never sees the text `loop`.

The assembler converts `loop` into a numeric offset.

The flow is:

```text
label in .s file
    -> assembler calculates offset
    -> raw .bin contains encoded branch instruction
    -> emulator decodes signed offset
    -> emulator computes target pc
```

## 4. Branch offsets are relative

ARM64 branch instructions usually store a signed offset relative to the current branch instruction.

Example:

```text
0x1000: b target
0x1004: nop
0x1008: hlt #0   <- target
```

The branch target is 8 bytes ahead of the branch instruction.

So the decoded offset is:

```text
+8
```

Then the emulator computes:

```text
target = current_pc + offset
target = 0x1000 + 8
target = 0x1008
```

Backward branches use negative offsets.

Example:

```text
0x1004: sub x0, x0, #1
0x1008: cbnz x0, loop
```

If `loop` is at `0x1004`, the branch at `0x1008` has offset:

```text
-4
```

So:

```text
target = 0x1008 - 4
target = 0x1004
```

That is how loops work.

## 5. `B` — unconditional branch

`B` always branches.

```asm
b target
```

Means:

```text
pc = target
```

It does not check registers or flags.

## 6. `CBZ` and `CBNZ`

`CBZ` means **compare and branch if zero**.

```asm
cbz x0, target
```

Means:

```text
if x0 == 0:
    pc = target
else:
    pc = pc + 4
```

`CBNZ` means **compare and branch if not zero**.

```asm
cbnz x0, target
```

Means:

```text
if x0 != 0:
    pc = target
else:
    pc = pc + 4
```

These instructions are great for simple loops.

## 7. Countdown loop example

Current v0.2 demo:

```asm
movz x0, #5
movz x1, #0
loop:
    add  x1, x1, #1
    sub  x0, x0, #1
    cbnz x0, loop
hlt #0
```

Meaning:

```text
x0 = 5   // countdown value
x1 = 0   // number of loop iterations

loop:
    x1 = x1 + 1
    x0 = x0 - 1
    if x0 != 0:
        go back to loop

stop
```

Final result:

```text
x0 = 0
x1 = 5
```

The loop body runs five times.

## 8. `CMP` and flags

`CMP` compares two values by subtracting them internally.

```asm
cmp x0, #5
```

Conceptually:

```text
result = x0 - 5
update NZCV flags
throw result away
```

`CMP` does not change `x0`.

It only changes flags.

## 9. What are NZCV flags?

The emulator stores four condition flags:

```text
N = negative
Z = zero
C = carry / no unsigned borrow
V = signed overflow
```

For v0.2, the most important one is `Z`.

If:

```asm
movz x0, #5
cmp  x0, #5
```

then:

```text
x0 - 5 = 0
Z = 1
```

If:

```asm
movz x0, #4
cmp  x0, #5
```

then:

```text
x0 - 5 != 0
Z = 0
N = 1, because the result is negative
```

## 10. `B.cond` — branch using flags

`B.cond` means branch if a condition is true.

Examples:

```asm
b.eq target
b.ne target
b.lt target
b.ge target
b.gt target
b.le target
```

These depend on NZCV flags.

Common conditions:

```text
EQ: equal                  Z == 1
NE: not equal              Z == 0
LT: signed less than       N != V
GE: signed greater/equal   N == V
GT: signed greater than    Z == 0 && N == V
LE: signed less/equal      Z == 1 || N != V
```

## 11. `CMP` + `B.EQ` example

```asm
movz x0, #7
cmp  x0, #7
b.eq equal
movz x1, #99
equal:
movz x2, #1
hlt  #0
```

Step by step:

```text
x0 = 7
compare x0 with 7
Z = 1 because they are equal
b.eq checks Z == 1
branch is taken
x1 = 99 is skipped
x2 = 1
halt
```

Final result:

```text
x0 = 7
x1 = 0
x2 = 1
Z = 1
```

## 12. Trace mode

v0.2 adds trace mode:

```sh
./emulator trace examples/v0_2/trace_loop.bin
```

It prints every `pc` before executing the instruction at that address.

Example shape:

```text
trace pc=0x0000000000001000
trace pc=0x0000000000001004
trace pc=0x0000000000001008
trace pc=0x0000000000001004
trace pc=0x0000000000001008
...
halted
```

Repeated addresses show that a loop is running.

## 13. What changed in the emulator code

v0.2 expands `cpu_decode()` so it recognizes more instruction shapes:

```text
B
B.cond
CBZ
CBNZ
CMP immediate
CMP register
```

It expands `cpu_step()` so instructions can update `pc` in two ways:

```text
normal instruction: pc = pc + 4
branch taken:       pc = target
```

It also adds flag updates for `CMP` and a condition helper for `B.cond`.

## 14. What is still not supported

v0.2 still does not support:

```text
load/store instructions
stack operations
function calls
ELF loading
Mach-O loading
syscalls
interactive debugger REPL
ADD register
```

The examples intentionally use `ADD immediate` so `ADD register` can stay deferred.

## 15. How to try it

Build everything:

```sh
make clean
make
make examples
```

Run the countdown loop:

```sh
./emulator run examples/v0_2/cbnz_countdown.bin
```

Run a compare + branch example:

```sh
./emulator run examples/v0_2/cmp_beq.bin
```

Run trace mode:

```sh
./emulator trace examples/v0_2/trace_loop.bin
```

## 16. One-sentence summary

v0.2 teaches the emulator how to stop moving forward blindly: branch instructions can set `pc` to another address, and `CMP` + flags let the program choose whether a conditional branch should be taken.
